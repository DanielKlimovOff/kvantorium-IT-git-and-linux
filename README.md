# kvantorium-IT-git-and-linux

Это занятие посвящено изучению базовых команд в терминале linux и работе с GitHub.

## Терминал Linux

> Терминал Linux (консоль, командная строка) — это текстовый интерфейс для прямого управления операционной системой с помощью команд. Он позволяет управлять файлами, устанавливать программы, настраивать систему и автоматизировать задачи. 

1. Откройте терминал

Чтобы открыть терминал можно комбинацией клавишь `Ctrl + Alt + T` или в меню приложений открыть терминал или консоль или bash.

2. Узнайте в какой директории вы находитесь

Чтобы узнать полный путь к текущей директории введите `pwd` которая так и расшифровывается pwd - print working directory.

3. Выведите содержимое текущей директории

Чтобы узнать содержимое текущей директории введите команду `ls`.

4. Узнайте размер файлов в текущей директории

Многие команды в терминале linux могут работать по разному, если передавать в них разные флаги. Флаги бывает короткие, перед которыми ставится один минус "-" и длинные с двуми минусами "--". Короткие флаги можно комбинировать между собой. 
- `ls -l` - флаг `-l` выводит содержимое в формате таблицы с дополнительными данными о владельце файла, времени последнего изменения, правах доступа и др.
- `ls -S` - флаг `-S` сортирует содержимое по размеру от наибольшего к наименьшему.
- `ls -h` - флаг `-h` выводит размеры файлов в читаемом для человека формате (кб, мб, гб).
- `ls --help` - флаг `--help` выводит памятку по использованию этой команды с описанием всех возможных флагов и параметров.

Таким образом чтобы узнать все размер всех файлов в директории можно вызвать команду.
```
ls -lhS
```

5. Создайте директорию my_new_dir

Чтобы создать новую директорию используйте команду `mkdir` и через пробел укажите путь к новой директории.

В Linux существуют 2 формата ввода путей к файлам:
- абсолютный - начиная от корневой директории / и до того файла или директории. Используется редко, так как при перемещении директории в другое место все абсолютные пути сломаются.
- относительный - начиная от текущей рабочей директории. Текущую рабочую директорию обозначают . Директорию на уровень выше обозначают .. 

- `mkdir my_new_dir` - создает my_new_dir в текущей директории
- `mkdir ./my_new_dir` - делает тоже самое
- `mkdir ../my_new_dir` - создает my_new_dir рядом с текущей директорией
- `mkdir /my_new_dir` - создает my_new_dir в корне файловой системы (наверняка у вас не получится создать новую папку в корне файловой системы без прав суперпользователя)

6. Смените текущую рабочую директорию на my_new_dir

Чтобы сменить рабочую директорию используется команда (как ни странно) `cd` - change directory. 

- `cd my_new_dir` - изменяет рабочую директорию на my_new_dir
- `cd` - без параметров эта команда устанавливает директорию в домашнюю директорию вашего пользователя 

7. Откройте VS Code

Чтобы запустить любое приложение из терминала, нам нужно просто ввести его имя.
- `code` - открывает VS Code
- `code .` - открывает VS Code в текущей директории
- `code my_file.txt` - открывает файл my_file.txt в VS Code 

8. Создайте файлы my_file1.txt и my_file2.txt

Можете сделать это любым удобным для вас способом. Вот как это можно сделать используя терминал: `touch my_file1.txt my_file2.txt` - создает 2 пустых файла в текущей директории

9. Запишите в эти файлы что-нибудь

Можете сделать это любым удобным для вас способом. Вот как это можно сделать используя терминал:

- `echo 'my name is daniel' > my_file1.txt`
- `echo 'some text' > my_file2.txt`

10. Удалите эти файлы и директорию my_new_dir

Чтобы удалить файл или директорию используется команда `rm`. Будьте осторожны с этой командой. Можно случайно удалить важные файлы или сломать свою систему.

- `rm my_new_dir/my_file1.txt my_new_dir/my_file2.txt` - удаляет 2 файла
- `rm -d my_new_dir` - удаляет директорию my_new_dir (флаг -d нужен для удаления пустой директорий)


## Репозиторий Git

> Git — это распределенная система управления версиями, созданная Линусом Торвальдсом в 2005 году для эффективного отслеживания изменений в файлах (преимущественно коде), работы над проектами в команде, возможности возврата к предыдущим версиям и безопасного экспериментирования. Она хранит «снимки» проекта, работает локально и часто используется с сервисами типа GitHub. 

1. Склонируйте репозиторий

```
git clone https://github.com/DanielKlimovOff/kvantorium-IT-git-and-linux.git
```

2. Перейдите в папку с репозиторием

```
cd kvantorium-IT-git-and-linux
```

3. Создайте свою ветку

```
git branch daniel_klimov
```

4. Переключитесь на свою ветку

```
git switch daniel_klimov
```

5. Запустите скрипт angry_tester.py
